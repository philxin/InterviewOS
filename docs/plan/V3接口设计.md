# API 接口设计 - V3

## 📋 概述

V3 接口设计围绕以下目标展开：

1. 将文件导入升级为可观测的文档索引任务
2. 提供最小可用的检索 API
3. 在不破坏 V2 主链路的前提下增强训练接口
4. 为自动知识点识别预留扩展接口

所有 API 继续遵循统一响应包结构：

**成功响应**

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

**错误响应**

```json
{
  "code": 400,
  "message": "error message",
  "data": null
}
```

**响应语义**

- 成功请求：使用 `2xx`，且 `code = 0`
- 失败请求：使用 `4xx/5xx`，且 `code` 与错误类型一致
- 禁止“业务失败仍返回 200”

---

## 🔗 基础信息

- **Base URL**: `http://localhost:8080/api`
- **Content-Type**: `application/json`
- **字符编码**: UTF-8
- **认证方式**: `Authorization: Bearer <token>`

除 `POST /api/auth/*` 等公开接口外，V3 新增接口全部要求登录访问。

---

## 🔄 V3 相比 V2 的主要变化

### 1. 文件导入接口升级

- V2 的 `/knowledge/file-imports` 只有任务开始与结果查询
- V3 增加文档 id、chunk 进度、部分失败与索引状态

### 2. 新增文档与检索接口

- 新增文档列表 / 文档详情
- 新增最小可用检索接口

### 3. 训练接口行为增强

- 保持 `POST /training/sessions`、`POST /training/sessions/{sessionId}/answers` 主路径不变
- 响应中可选返回本题引用依据
- 历史详情中可选返回引用快照

### 4. 自动知识点识别接口延后

- P1 再开放概念候选查询与接受接口
- P0 不要求前端依赖此能力

## 🧱 枚举约定

### KnowledgeFileImportStatus

| 值 | 说明 |
|----|------|
| `PENDING` | 任务已创建，等待处理 |
| `PROCESSING` | 文件解析中 |
| `CHUNKING` | 文本切片中 |
| `EMBEDDING` | 向量化中 |
| `SUCCESS` | 全部成功 |
| `PARTIAL` | 部分 chunk 成功，部分失败 |
| `FAILED` | 整体失败 |

### RetrievalMode

| 值 | 说明 |
|----|------|
| `NONE` | 未启用检索 |
| `RAG` | 启用检索且命中有效引用 |
| `FALLBACK` | 启用检索但因低命中或异常降级 |

### ReferenceUsageType

| 值 | 说明 |
|----|------|
| `QUESTION` | 题目生成使用的引用 |
| `HINT` | 提示生成使用的引用 |
| `FEEDBACK` | 反馈生成使用的引用 |

## 1. 文件导入接口

### 1.1 创建文件导入任务

```http
POST /api/knowledge/file-imports
Content-Type: multipart/form-data
```

**请求参数**

- `file`: 必填，支持 `txt / md / pdf`
- `defaultTags`: 可选，逗号分隔

**响应示例（HTTP 202）**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "importId": "0f0b8df4-95af-4f23-a7fc-086b3f4a1b92",
    "documentId": null,
    "status": "PENDING",
    "fileName": "java-interview-notes.pdf",
    "fileSize": 125388,
    "totalChunks": 0,
    "embeddedChunks": 0,
    "failedChunks": 0,
    "createdAt": "2026-03-24T20:00:00"
  }
}
```

**错误语义**

- `400`: 文件为空、扩展名不支持
- `401`: 未登录
- `413`: 文件过大
- `429`: 导入任务过于频繁

### 1.2 查询文件导入任务

```http
GET /api/knowledge/file-imports/{importId}
```

**响应示例（HTTP 200）**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "importId": "0f0b8df4-95af-4f23-a7fc-086b3f4a1b92",
    "documentId": "2cf12d0f-e00c-4fbe-b858-4f4685a62844",
    "status": "EMBEDDING",
    "fileName": "java-interview-notes.pdf",
    "fileSize": 125388,
    "totalChunks": 32,
    "embeddedChunks": 21,
    "failedChunks": 0,
    "defaultTags": [
      "java",
      "backend"
    ],
    "failureReason": null,
    "createdAt": "2026-03-24T20:00:00",
    "updatedAt": "2026-03-24T20:00:12",
    "completedAt": null
  }
}
```

**说明**

- `documentId` 在文档创建前允许为 `null`
- `PARTIAL` 状态下前端应提示“文档可用，但部分切片未建立索引”

## 2. 文档接口

### 2.1 获取文档列表

```http
GET /api/knowledge/documents?page=1&size=20&status=ACTIVE
```

**响应字段建议**

- `documentId`
- `title`
- `originalFileName`
- `status`
- `totalChunks`
- `activeChunks`
- `indexedAt`
- `updatedAt`

### 2.2 获取文档详情

```http
GET /api/knowledge/documents/{documentId}
```

**响应字段建议**

- 基础元信息
- 索引状态
- 最近一次导入任务 id
- chunk 统计
- 来源元信息摘要

**错误语义**

- `404`: 当前用户无权访问该文档或文档不存在

---

## 3. 检索接口

### 3.1 检索知识片段

```http
GET /api/rag/search?query=Spring%20Boot%20自动配置原理&topK=5
```

**请求参数**

- `query`: 必填，长度 `1-500`
- `topK`: 可选，默认 `5`，最大 `10`
- `documentId`: 可选，仅在单文档内检索

**响应示例（HTTP 200）**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "query": "Spring Boot 自动配置原理",
    "topK": 5,
    "hitCount": 3,
    "degraded": false,
    "items": [
      {
        "chunkId": 1001,
        "documentId": "2cf12d0f-e00c-4fbe-b858-4f4685a62844",
        "documentTitle": "java-interview-notes",
        "score": 0.8921,
        "snippet": "自动配置的核心是基于条件装配和 spring.factories / AutoConfiguration.imports ...",
        "pageFrom": 3,
        "pageTo": 3
      }
    ]
  }
}
```

**错误语义**

- `400`: query 为空或超长
- `401`: 未登录
- `404`: 指定 `documentId` 不属于当前用户

## 4. 训练接口增强

### 4.1 启动训练会话

保持 V2 主路径：

```http
POST /api/training/sessions
```

**V3 行为变化**

- 后端可在出题前执行检索
- 如果检索命中有效 chunk，则按固定引用集生成题目
- 如果低命中，则走 `FALLBACK`

**响应新增字段建议**

```json
{
  "retrievalMode": "RAG",
  "references": [
    {
      "usageType": "QUESTION",
      "documentTitle": "java-interview-notes",
      "excerpt": "自动配置通过条件注解控制 Bean 是否装配",
      "pageFrom": 3,
      "pageTo": 3
    }
  ]
}
```

### 4.2 获取题目提示

保持 V2 主路径：

```http
POST /api/training/sessions/{sessionId}/questions/{questionId}/hint
```

**V3 行为变化**

- 提示生成可使用当前题目的固定引用集或单独的 hint 引用集
- 响应中可附带 `references`

### 4.3 提交回答

保持 V2 主路径：

```http
POST /api/training/sessions/{sessionId}/answers
```

**V3 行为变化**

- 反馈生成可附带 `retrievalMode`
- 反馈响应可返回 `references`

**响应新增字段建议**

```json
{
  "retrievalMode": "RAG",
  "references": [
    {
      "usageType": "FEEDBACK",
      "documentTitle": "java-interview-notes",
      "excerpt": "回答自动配置时需要覆盖条件装配、SPI 装载入口、覆盖机制",
      "pageFrom": 3,
      "pageTo": 4
    }
  ]
}
```

### 4.4 获取训练详情

保持 V2 主路径：

```http
GET /api/training/sessions/{sessionId}
```

**V3 行为变化**

- 题目详情可包含 `references`
- 便于结果页解释“本题依据了哪些材料”

## 5. P1 预留接口

### 5.1 获取文档候选知识点

```http
GET /api/knowledge/documents/{documentId}/concepts
```

### 5.2 接受候选知识点

```http
POST /api/knowledge/concepts/{conceptId}/accept
```

**用途**

- 将候选概念转为正式 `Knowledge`
- 接受时可自定义标题、标签、内容摘要

## 🔐 安全约定

### 1. 资源隔离

- 文档、chunk、检索结果全部按当前登录用户隔离
- 非 owner 访问时返回 `404` 或 `403`

### 2. 检索安全

- 进入 prompt 的检索片段必须被标记为“引用材料”
- 不允许将检索文本当作系统指令执行
- 返回前端的片段只给摘要，不返回整个文档原文

### 3. 限流与防滥用

- 文件导入接口建议增加用户级频率限制
- 检索接口建议限制 `topK` 和 query 长度
- 训练反馈接口需限制答案长度与调用频率

---

## ✅ V3 接口完成标准

- 文件导入接口可返回阶段化进度
- 文档与检索接口可满足最小前端接入
- 训练主路径不改签名即可接入 retrieval context
- 历史与结果页可获取训练引用快照
- 接口层安全语义与 HTTP Status 保持正确
