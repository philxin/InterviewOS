# 数据库设计 - V1

## 📊 概述

V1 使用 PostgreSQL 作为主数据库，采用简单的两表设计：

- `knowledge` - 知识点表
- `training_record` - 训练记录表

---

## 📋 表结构设计

### 表一：knowledge（知识点表）

存储用户创建的各个知识点，记录当前的掌握程度。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PRIMARY KEY, AUTO INCREMENT | 主键 |
| title | varchar(200) | NOT NULL | 知识点标题 |
| content | text | NOT NULL | 知识点详细内容 |
| mastery | int | NOT NULL, DEFAULT 0 | 掌握度 (0-100) |
| created_at | timestamp | NOT NULL, DEFAULT NOW() | 创建时间 |

**索引**：
- `idx_created_at`: created_at DESC（用于按创建时间排序）

**业务规则**：
- `mastery` 取值范围 0-100
- `mastery` 更新公式：`newMastery = oldMastery * 0.7 + overall * 0.3`

---

### 表二：training_record（训练记录表）

存储每次训练的完整记录，包括问题、回答和 AI 评分结果。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PRIMARY KEY, AUTO INCREMENT | 主键 |
| knowledge_id | bigint | NOT NULL, FOREIGN KEY | 关联知识点 ID |
| question | text | NOT NULL | AI 生成的问题 |
| answer | text | NOT NULL | 用户的回答 |
| accuracy | int | NOT NULL | 准确度评分 (0-100) |
| depth | int | NOT NULL | 深度评分 (0-100) |
| clarity | int | NOT NULL | 清晰度评分 (0-100) |
| overall | int | NOT NULL | 综合评分 (0-100) |
| strengths | text | NULL | 优点描述 |
| weaknesses | text | NULL | 缺点描述 |
| suggestions | text | NULL | 建议 (JSON 数组字符串) |
| example_answer | text | NULL | 示例回答 |
| created_at | timestamp | NOT NULL, DEFAULT NOW() | 创建时间 |

**外键**：
- `fk_knowledge_id`: knowledge_id → knowledge(id)

**索引**：
- `idx_knowledge_created`: knowledge_id, created_at DESC（用于查询某知识点的训练历史）
- `idx_created_at`: created_at DESC（用于查询所有训练历史）

**业务规则**：
- 所有评分字段取值范围 0-100
- `overall` = (accuracy + depth + clarity) / 3（取整）

## 📝 SQL 创建脚本

```sql
-- 创建知识点表
CREATE TABLE knowledge (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    mastery INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 创建训练记录表
CREATE TABLE training_record (
    id BIGSERIAL PRIMARY KEY,
    knowledge_id BIGINT NOT NULL,
    question TEXT NOT NULL,
    answer TEXT NOT NULL,
    accuracy INTEGER NOT NULL,
    depth INTEGER NOT NULL,
    clarity INTEGER NOT NULL,
    overall INTEGER NOT NULL,
    strengths TEXT,
    weaknesses TEXT,
    suggestions TEXT,
    example_answer TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    CONSTRAINT fk_knowledge_id
        FOREIGN KEY (knowledge_id)
        REFERENCES knowledge(id)
        ON DELETE CASCADE
);

-- 创建索引
CREATE INDEX idx_knowledge_created ON training_record(knowledge_id, created_at DESC);
CREATE INDEX idx_created_at_tr ON training_record(created_at DESC);
CREATE INDEX idx_created_at_kn ON knowledge(created_at DESC);

-- 添加检查约束
ALTER TABLE knowledge ADD CONSTRAINT chk_mastery_range
    CHECK (mastery >= 0 AND mastery <= 100);

ALTER TABLE training_record ADD CONSTRAINT chk_accuracy_range
    CHECK (accuracy >= 0 AND accuracy <= 100);

ALTER TABLE training_record ADD CONSTRAINT chk_depth_range
    CHECK (depth >= 0 AND depth <= 100);

ALTER TABLE training_record ADD CONSTRAINT chk_clarity_range
    CHECK (clarity >= 0 AND clarity <= 100);

ALTER TABLE training_record ADD CONSTRAINT chk_overall_range
    CHECK (overall >= 0 AND overall <= 100);
```

## 🔍 常用查询示例

### 1. 查询所有知识点（按创建时间倒序）

```sql
SELECT * FROM knowledge
ORDER BY created_at DESC;
```

### 2. 查询某知识点的训练历史

```sql
SELECT * FROM training_record
WHERE knowledge_id = ?
ORDER BY created_at DESC;
```

### 3. 查询所有训练记录（按时间倒序）

```sql
SELECT tr.*, k.title as knowledge_title
FROM training_record tr
JOIN knowledge k ON tr.knowledge_id = k.id
ORDER BY tr.created_at DESC;
```

### 4. 统计某知识点的平均掌握度

```sql
SELECT AVG(overall) as avg_score, COUNT(*) as training_count
FROM training_record
WHERE knowledge_id = ?;
```

### 5. 查找低掌握度知识点（需要加强训练）

```sql
SELECT * FROM knowledge
WHERE mastery < 50
ORDER BY mastery ASC, created_at DESC;
```

## 📊 V1 数据量估算

假设单用户使用 6 个月：

- **知识点数量**: 20-50 个
- **每知识点训练次数**: 5-10 次
- **训练记录总数**: 100-500 条

**存储空间**：
- knowledge 表: < 100 KB
- training_record 表: < 5 MB

V1 不需要考虑数据量优化，简单的设计即可满足需求。

## 🚀 V2 扩展预留

虽然 V1 不使用，但表结构预留了扩展空间：

1. **用户系统**: 增加 `user_id` 字段
2. **向量检索**: 增加 `embedding` 字段（pgvector）
3. **标签系统**: 增加 `tags` 字段或关联表
4. **多轮对话**: 增加 `session_id` 和 `turn_number` 字段
