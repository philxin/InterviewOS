# InterviewOS V1 版本规划

## 📋 版本概述

V1 基线版本是一个 **最小可用产品 (MVP)**，专注于实现完整的训练闭环：

> **创建知识点 → AI 提问 → 用户回答 → AI 评分 → 更新掌握度 → 查看历史记录**

---

## 🎯 核心目标

### 功能目标
- [x] 知识点管理（增删改查）
- [ ] AI 驱动的问题生成
- [ ] AI 驱动的答案评分
- [ ] 掌握度自动更新
- [ ] 训练历史记录查看

### 技术目标
- [ ] 建立清晰的分层架构
- [ ] 实现 LLM 抽象层
- [ ] 统一异常处理和返回格式
- [ ] 完整的前后端联调

---

## 🚫 V1 明确不做

在 V1 完成前，**禁止**以下功能：

| 功能 | 说明 | 计划版本 |
|------|------|----------|
| 用户系统 | 注册、登录、权限 | V2 |
| 向量数据库 | pgvector 语义检索 | V2 |
| Redis 缓存 | 训练缓存、会话管理 | V2 |
| 多轮对话 | 追问、讨论模式 | V3 |
| 复杂统计 | 能力雷达图、成长曲线 | V3 |
| 知识图谱 | 知识点关联 | V3 |
| 文件导入 | PDF/Word 解析 | V2 |

## ✅ V1 完成标准

当满足以下条件时，V1 视为完成：

1. **功能完整性**
   - 可以创建、编辑、删除知识点
   - 可以发起训练并获得 AI 问题
   - 可以提交答案并获得 AI 评分
   - 掌握度按公式正确更新
   - 可以查看训练历史

2. **技术质量**
   - 所有接口返回统一格式
   - 无业务逻辑在 Controller
   - LLM 调用完全抽象
   - 日志记录完整

3. **用户体验**
   - 无前端控制台错误
   - 操作流程清晰
   - 错误提示友好

## 📈 从 V1 到 V2 的演进路径

```
V1 (当前)
  ↓ 单人本地训练
  ↓
V2
  ├─ 用户系统 (多用户隔离)
  ├─ 文件导入 (知识来源扩展)
  └─ 向量检索 (相关问题推荐)
  ↓
V3
  ├─ 多轮对话 (深度训练)
  ├─ 知识图谱 (体系化学习)
  └─ 能力模型 (多维度评估)
```
