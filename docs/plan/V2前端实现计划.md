# InterviewOS V2 前端实现计划

## 📁 目标

V2 前端实现目标是在现有 Vue 3 + TypeScript + Pinia + Vue Router 架构上完成以下升级：

1. 支持用户登录与登录态管理
2. 让用户能更快进入训练，而不是停留在知识点管理页
3. 适配新的 V2 训练接口与结果结构
4. 重构结果页，让反馈更易理解、更可执行
5. 为后续首页薄弱项推荐、今日训练包等能力预留页面空间

---

## 🧭 实现原则

### 1. 保留现有技术栈与基础结构

- 继续使用 Vue 3 + TypeScript + Vite
- 继续使用 Pinia 做状态管理
- 继续使用 Axios + 统一客户端拦截器
- 不引入 UI 框架

### 2. 先打通主链路，再做视觉增强

- P0 先保证：登录 → 岗位选择 → 知识导入 → 开始训练 → 提交回答 → 查看结果
- P1 再补：更完整的首页概览、提示交互、页面质感优化
- P2 再补：推荐训练包、薄弱项首页入口、回练提醒

### 3. 页面设计服务“短时高频训练”

- 首页优先呈现行动入口，不做纯展示页
- 训练页优先保障低干扰、高专注
- 结果页优先给出可理解判断，不先铺满分析文字

### 4. 与 V2 接口契约严格对齐

- 以前端类型定义为接口文档的镜像，不自行发明字段
- 所有 V2 字段变更以 [API接口设计.md](D:\InterviewOS\docs\plan\API接口设计.md) 为准

## 4. 路由与页面流转

### 改造 `src/router/index.ts`

建议路由：

```ts
/
/login
/onboarding
/knowledge/new
/knowledge/edit/:id
/knowledge/import
/training/session/:sessionId
/result/:sessionId
/history
```

### 路由守卫规则

1. 未登录访问受保护页，跳转 `/login`
2. 已登录但未完成岗位选择，优先跳转 `/onboarding`
3. 登录后访问 `/login`，跳转首页

### 页面流转主链路

```text
登录
  ↓
岗位选择
  ↓
首页 / 知识导入
  ↓
开始训练
  ↓
训练页
  ↓
结果页
  ↓
返回首页或历史页
```

## 5. 页面级实现计划

### 5.1 `LoginView.vue`

#### 功能

- 登录
- 注册切换
- 登录错误提示
- 成功后写入 token 与 user

#### 重点

- 表单验证简洁明确
- 登录成功后按 `targetRole` 决定跳首页还是 onboarding

---

### 5.2 `OnboardingView.vue`

#### 功能

- 选择目标岗位
- 提交训练方向
- 完成后跳转首页

#### 重点

- 只做轻量岗位选择
- 不做简历深度解析

---

### 5.3 `DashboardView.vue`

#### V2 目标

首页不再只是“知识点列表”，而是“训练入口 + 知识点管理”的结合页。

#### P0 功能

- 显示知识点列表
- 支持按标签筛选
- 默认隐藏已归档知识点
- 支持跳转新建、编辑、批量导入
- 支持开始训练

#### P1 功能

- 显示薄弱项摘要
- 显示最近训练记录

#### P2 功能

- 今日推荐训练包入口

---

### 5.4 `KnowledgeFormView.vue`

#### V2 功能

- 标题
- 内容
- 标签输入
- 保存 / 取消

#### 重点

- 标签录入体验要足够轻量
- 保持页面简洁，不做复杂富文本编辑器

---

### 5.5 `KnowledgeImportView.vue`

#### P0 功能

- 批量粘贴或结构化输入知识点
- 一次导入多条
- 展示失败项

失败项展示至少包含：

- 原始序号
- 标题
- 失败原因

#### P1 功能

- 文件上传导入
- 展示导入任务状态

#### 重点

- 这是 V2 降低录入成本的关键页面
- 优先保证批量导入可用
- 前端校验需与后端密码/标签/批量导入规则保持一致

---

### 5.6 `TrainingSessionView.vue`

#### V2 目标

训练页强调专注、节奏和完成率。

#### 功能

- 显示题目类型与难度
- 显示问题正文
- 输入回答
- 提交回答
- 获取提示（P1）

#### 重点

- 加载状态清晰
- 提示项只给思路，不直接给答案
- 页面不堆信息，避免干扰回答

---

### 5.7 `ResultView.vue`

#### V2 结果结构

页面信息层级应为：

1. 档位与主要问题
2. 缺失点
3. 更好回答思路
4. 自然参考表达

#### 功能

- 展示 `band`
- 展示 `majorIssue`
- 展示 `missingPoints`
- 展示 `betterAnswerApproach`
- 展示 `naturalExampleAnswer`
- 展示掌握度前后变化

#### 重点

- 不再以旧三维评分条作为主视觉
- 结果要短、清楚、可改进

---

### 5.8 `HistoryView.vue`

#### 功能

- 展示训练历史列表
- 支持按知识点筛选
- 点击进入单次训练详情 / 结果回看
- 支持分页

#### 重点

- 优先展示对用户有帮助的字段：
  - 时间
  - 题型
  - 难度
  - 档位
  - 主要问题

## 7. UI 与交互方向

### 首页

- 从纯列表页转向行动入口页
- 用户一眼能看到“下一步做什么”

### 训练页

- 减少视觉噪音
- 聚焦问题与回答
- 保持按钮与状态反馈明确

### 结果页

- 先结论，后分析
- 先短文本，后详细内容
- 示例表达使用更接近日常面试口吻的呈现

### 全局体验

- 去掉明显的 Demo 感
- 颜色与版式服务“训练产品”而不是通用后台
- 保留移动端基本可用性

## ⚠️ 风险点

### 风险 1：前端字段结构与后端 DTO 不一致

- 处理方式：类型定义完全镜像 [API接口设计.md](D:\InterviewOS\docs\plan\API接口设计.md)

### 风险 2：登录态与路由守卫切换时出现循环跳转

- 处理方式：优先理清 `未登录 / 已登录未 onboarding / 已登录已 onboarding` 三种状态

### 风险 3：ResultView 仍沿用旧评分思路

- 处理方式：先删除旧三维中心结构，再围绕档位反馈重建布局

### 风险 4：批量导入页过重

- 处理方式：P0 只做结构化批量录入，文件导入放到 P1
