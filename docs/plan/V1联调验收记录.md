## 4. 手工联调检查单（你配置服务器后执行）

1. 打开前端首页，创建知识点，检查列表显示与掌握度徽章。
2. 进入训练页，触发 `POST /api/training/start` 并收到问题文本。
3. 提交回答，检查 `POST /api/training/submit` 返回评分与建议。
4. 返回首页，确认知识点 `mastery` 变化。
5. 打开历史页，确认 `GET /api/training/history` 与筛选行为正确。
6. 用非法参数访问接口，确认前后端错误提示与 HTTP 状态码匹配。

## 6. 服务器配置清单（待你填充）

### 6.1 后端环境变量（Spring Boot）

建议按下面模板在服务器环境中配置：

```env
# 运行模式
SPRING_PROFILES_ACTIVE=dev-postgres
APP_PRIMARY_DATASOURCE=postgresql

# 服务地址
SERVER_PORT=8080
SERVER_CONTEXT_PATH=/api

# PostgreSQL（V1 单库）
POSTGRES_URL=jdbc:postgresql://<host>:5432/<db_name>
POSTGRES_USERNAME=<username>
POSTGRES_PASSWORD=<password>

# Redis
REDIS_HOST=<host>
REDIS_PORT=6379
REDIS_PASSWORD=<password_or_empty>
REDIS_DATABASE=0

# OpenAI / LLM
OPENAI_API_KEY=<your_api_key>
OPENAI_BASE_URL=https://api.openai.com
OPENAI_CHAT_MODEL=gpt-4o-mini
OPENAI_CHAT_TEMPERATURE=0.7

# JPA
JPA_DDL_AUTO=update
JPA_SHOW_SQL=false
JPA_FORMAT_SQL=true
DB_TIME_ZONE=UTC

# 日志
APP_LOG_LEVEL=INFO
SPRING_AI_LOG_LEVEL=INFO
SPRING_WEB_LOG_LEVEL=INFO
```

## 8. 密钥加密方案（可选增强）

当前建议基线：`Profile + 环境变量`（已支持）。  
如果你需要更高安全等级，可选以下方案：

1. Secret Manager（推荐）  
   例如 HashiCorp Vault、AWS Secrets Manager、阿里云 KMS/参数存储。应用启动时按权限拉取，不落盘明文。
2. 加密文件托管（GitOps 场景）  
   使用 `SOPS + age/KMS` 管理加密后的 `*.enc.yaml`，部署时解密注入环境。
3. Jasypt（过渡方案）  
   在配置中使用 `ENC(...)`，解密主密钥仍需通过环境变量注入；适合没有 Secret Manager 的过渡阶段。

注意：`frontend` 的 `VITE_*` 变量会被打包到浏览器端，不可存任何 secret。
