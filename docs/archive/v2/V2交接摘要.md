# InterviewOS V2 交接摘要

## 1. 项目定位

InterviewOS `V2` 的目标是将 V1 的“单人训练闭环”升级为“可持续使用的面试训练系统”。

V2 重点解决：

1. 用户体系与数据隔离
2. 训练方向初始化
3. 知识点批量导入与文件导入任务化
4. 更真实的训练模型与更可信的反馈结构
5. 首页概览、推荐训练与回练闭环

## 2. 当前技术栈

后端：

- Java 21
- Spring Boot 3.3.6
- Spring AI 1.0.0-M5
- Spring Web / Data JPA / Validation / Security / Redis
- PostgreSQL
- pgvector 依赖已在工程中引入，但 V2 主链路未正式使用

前端：

- Vue 3
- TypeScript
- Vite
- Pinia
- Vue Router
- Tailwind CSS
- Axios
- ECharts

## 3. V2 已完成内容

### 后端

1. 用户体系：`AppUser`、注册 / 登录 / `me`、JWT 鉴权
2. 用户训练方向：`targetRole` 与 onboarding 接口
3. 知识点 owner 模型：`Knowledge`、`KnowledgeTag`、按用户隔离查询
4. 批量导入：`POST /knowledge/batch-import`
5. 文件导入任务化：`knowledge_file_import`、异步解析 `txt / md / pdf`
6. 训练主模型：`TrainingSession + TrainingQuestion`
7. 提示接口：`POST /training/sessions/{sessionId}/questions/{questionId}/hint`
8. V2 反馈结构：`score / band / majorIssue / missingPoints / betterAnswerApproach / naturalExampleAnswer`
9. 首页概览：`/dashboard/overview`
10. 今日推荐训练包、自动组题、回练提醒

### 前端

1. 登录、登出、token 持久化与路由守卫
2. onboarding 页面与登录后主链路切换
3. 知识点标签编辑、批量导入、文件导入任务页
4. 多题 session 训练页、结果页、历史页
5. hint 交互、首页概览、推荐入口、回练入口
6. 空态 / 错误态统一组件化

### 文档

1. V2 版本规划、数据库设计、接口设计、前后端实现计划已冻结
2. V2 执行看板与联调回归记录已收口
3. V2 迁移策略已冻结
4. 当前 V2 文档已整体归档到 `docs/archive/v2/`

## 4. V2 核心业务规则

1. 所有训练与知识数据默认按当前登录用户隔离
2. 训练主模型以 `training_session + training_question` 为准，不再以 `training_record` 承载主链路
3. 反馈档位由服务端按 `score` 做确定性映射，不由 LLM 自由决定
4. 文件导入在 V2 中仍然落为单条 `Knowledge`，未拆分 document / chunk
5. V2 不引入 RAG、向量检索与自动知识点识别

## 5. 当前核心接口

认证与用户：

1. `POST /api/auth/register`
2. `POST /api/auth/login`
3. `GET /api/auth/me`
4. `PATCH /api/users/me/onboarding`

知识点：

1. `GET /api/knowledge`
2. `GET /api/knowledge/{id}`
3. `POST /api/knowledge`
4. `PUT /api/knowledge/{id}`
5. `DELETE /api/knowledge/{id}`
6. `POST /api/knowledge/batch-import`
7. `POST /api/knowledge/file-imports`
8. `GET /api/knowledge/file-imports/{importId}`

训练：

1. `POST /api/training/sessions`
2. `POST /api/training/sessions/{sessionId}/answers`
3. `POST /api/training/sessions/{sessionId}/questions/{questionId}/hint`
4. `GET /api/training/sessions`
5. `GET /api/training/sessions/{sessionId}`
6. `GET /api/training/recommendations/today`

Dashboard：

1. `GET /api/dashboard/overview`
2. `GET /api/dashboard/review-reminders`

默认 Base URL：

```text
http://localhost:8080/api
```

## 6. 当前配置与结构结论

1. V2 当前推荐数据库方案仍是 PostgreSQL 单库
2. Spring Security 与 JWT 已进入主链路，除认证接口外默认要求登录
3. `KnowledgeFileImport` 已具备任务模型，但尚未升级为 V3 的 document / chunk pipeline
4. `Spring AI + pgvector` 已可作为 V3 演进基础，不需要从零起步

## 7. 当前安全与可观测性结论

1. 登录态通过 Bearer Token 传递
2. owner 级资源访问控制已进入服务主链路
3. 日志默认脱敏，不打印密码与明文 token
4. LLM 日志记录长度、指纹、耗时，不直接打印正文
5. 文件导入已具备任务状态与失败原因摘要，但尚无 V3 级别检索 trace

## 8. 联调与验收结论

已确认通过：

1. 注册、登录、`me`
2. 用户 owner 隔离
3. 知识点 CRUD、标签与批量导入
4. 新训练主模型、多题 session、结果页与历史页
5. hint、dashboard、推荐训练与回练提醒
6. 文件导入任务化

当前结论：

- V2 已完成并归档
- 当前自然下一阶段是 V3：文件级知识库、检索增强训练与引用快照

## 9. 已知边界与后续建议

仍需关注：

1. V2 文件导入最终仍是单条 `Knowledge`，无法支撑细粒度检索
2. V2 评分仍主要依赖 `Knowledge.content`，缺少引用可追溯性
3. V2 尚未建立 document / chunk / vector 数据层

后续自然方向：

1. 升级文件导入 pipeline 为 document / chunk / embedding
2. 在训练题目、提示、反馈中引入 retrieval context
3. 记录训练引用快照，提升结果可解释性
4. 在 P1 再增加自动知识点识别能力
