# InterviewOS V2 归档

## 1. 归档说明

本文档用于冻结 InterviewOS `V2` 阶段的范围、交付物、验收结果和后续阅读入口。

归档原则：

- 将 `docs/plan` 下的 V2 文档迁移到 `docs/archive/v2/plan/`
- `docs/plan` 仅保留当前活跃版本 V3 文档
- 通过归档索引统一收敛 V2 的最终状态
- 后续如进入 V3 实施阶段，不再直接修改 V2 已冻结结论

归档时间：

- `2026-03-24`

V2 当前状态：

- 已完成
- 已完成文档收口与归档
- 可作为 V3 设计与新会话交接的稳定基线

## 2. V2 范围结论

V2 已完成能力：

1. 用户注册、登录、JWT 鉴权与数据隔离
2. 用户训练方向初始化
3. 知识点 owner 化、标签体系与批量导入
4. `training_session + training_question` 新训练主模型
5. 更可解释的反馈结构、提示能力与首页概览
6. 文件导入任务化、今日推荐训练包、自动组题、回练提醒
7. 前后端 V2 主链路联调、回归与文档冻结

V2 不包含：

1. RAG / 向量检索主链路
2. 自动知识点识别
3. 跨用户共享知识库
4. 多轮对话
5. 通用文档问答平台

## 4. 代码与运行基线

运行基线：

1. 后端：Java 21 + Spring Boot 3.3.6
2. 前端：Vue 3 + TypeScript + Vite
3. 默认 API Base URL：`http://localhost:8080/api`
4. 当前主业务数据库路径：PostgreSQL
5. V2 结束时已具备 `Spring AI + pgvector` 依赖，但未正式进入 RAG 主链路

## 5. 归档后的使用方式

如果要开启新的对话或新阶段工作，建议：

1. 先把 [V2 交接摘要](./V2交接摘要.md) 发给新的助手作为上下文
2. 如果涉及 V2 接口兼容，再补充 [API 接口设计](./plan/API接口设计.md)
3. 如果涉及 V2 训练模型或数据结构，再补充 [V2 数据库设计](./plan/V2数据库设计.md)
4. 如果进入 V3，请优先阅读 `docs/plan/` 下的 V3 文档，而不是继续修改 V2 归档文件
