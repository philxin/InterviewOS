# InterviewOS V2 迁移策略冻结

## 1. 目标

冻结 V1 旧表 `training_record` 到 V2 新模型（`training_session` + `training_question`）的上线策略，解除发布阻塞项 `BLK-02`。

冻结日期：`2026-03-18`

---

## 2. 环境策略

### 2.1 开发环境

- 允许重建数据库，不强制保留 V1 历史数据。
- 仅执行结构迁移脚本：
  - `scripts/postgres-v2-migration.sql`

### 2.2 测试环境

- 默认执行结构迁移脚本。
- 若需要验证历史数据兼容性，再执行可选回填脚本：
  - `scripts/postgres-v2-training-record-backfill.sql`

### 2.3 生产环境

- 先做结构迁移，再按业务需要决定是否回填历史 `training_record`。
- 不再将 `training_record` 作为 V2 主链路读写来源；V2 统一使用 `training_session` + `training_question`。
- 旧表保留为历史归档源，待一个稳定版本窗口后再评估清理。

---

## 3. SQL 执行顺序

1. 备份数据库（全库备份 + 核心表行数快照）。
2. 执行结构迁移：

```bash
export PGPASSWORD='<db-password>'
psql -h 127.0.0.1 -U interviewos -d interviewos -f scripts/postgres-v2-migration.sql
```

3. 如需保留 V1 历史训练可查询能力，再执行回填：

```bash
export PGPASSWORD='<db-password>'
psql -h 127.0.0.1 -U interviewos -d interviewos -f scripts/postgres-v2-training-record-backfill.sql
```

4. 执行回填核对：

```sql
SELECT COUNT(*) AS legacy_count FROM training_record;
SELECT COUNT(*) AS migrated_count FROM training_record_migration_map;
SELECT COUNT(*) AS session_count FROM training_session;
SELECT COUNT(*) AS question_count FROM training_question;
```

---

## 4. 回填口径

- 一条 `training_record` 回填为：
  - 一条 `training_session`（`COMPLETED`、单题）
  - 一条 `training_question`（`order_no = 1`）
- `question_type` 固定为 `FUNDAMENTAL`，`difficulty` 固定为 `MEDIUM`（V1 无对应字段）。
- `overall` 映射到 V2 `score` 与 `feedback_band`：
  - `<40`: `UNCLEAR`
  - `<55`: `INCOMPLETE`
  - `<70`: `BASIC`
  - `<85`: `GOOD`
  - `>=85`: `STRONG`
- 回填脚本幂等，重复执行不会重复写入同一条历史记录（依赖 `training_record_migration_map`）。

---

## 5. 上线回滚建议

- 若迁移后应用启动异常或关键接口验收失败：
  1. 停止新版本应用进程
  2. 回滚到迁移前备份
  3. 恢复上一版后端包并重启
- 迁移窗口期间，避免并发执行多份回填脚本。
