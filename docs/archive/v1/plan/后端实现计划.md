### 1. 通用层 (common/)

#### Result.java
```java
public class Result<T> {
    private int code;        // 0=成功, 非0=失败
    private String message;  // 提示信息
    private T data;          // 业务数据

    // 静态工厂方法
    public static <T> Result<T> success(T data)
    public static <T> Result<T> success()
    public static <T> Result<T> error(int code, String message)
}
```

#### BusinessException.java
```java
public class BusinessException extends RuntimeException {
    private int code;

    public BusinessException(String message)
    public BusinessException(int code, String message)
}
```

```java

public class GlobalExceptionHandler {
    @ExceptionHandler(BusinessException.class)
```

#### TrainingService.java
```java
@Service
public class TrainingService {
    // 开始训练：生成问题
    public String startTraining(Long knowledgeId);

    // 提交回答：评分并更新掌握度
    public EvaluationResult submitAnswer(
        Long knowledgeId,
        String question,
        String answer
    );

    // 获取训练历史
    public List<TrainingRecord> getHistory(Long knowledgeId);

    // 获取所有训练记录
    public List<TrainingRecord> getAllHistory();

    // 更新掌握度（核心公式）
    private void updateMastery(Knowledge knowledge, int overall) {
        int newMastery = (int)(knowledge.getMastery() * 0.7 + overall * 0.3);
        knowledge.setMastery(newMastery);
    }
}
```

### 7. 配置类

#### SecurityConfig.java
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .authorizeHttpRequests(auth -> auth
                .anyRequest().permitAll()  // V1 放行所有请求
            );
        return http.build();
    }
}
```

---

## ⚠️ 架构约束

1. **禁止在 Controller 写业务逻辑** - 只接收参数，调用 Service
2. **禁止直接在 Service 调用 HTTP** - 必须通过 LLMService 抽象
3. **禁止魔法字符串** - 所有常量定义在专用类中
4. **禁止在 Controller 写 try-catch** - 让全局异常处理器处理
5. **必须记录日志** - LLM 调用必须记录 prompt 和响应
