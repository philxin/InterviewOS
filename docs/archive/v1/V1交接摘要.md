# InterviewOS V1 交接摘要

## 1. 项目定位

InterviewOS 是一个面向面试训练场景的 AI 驱动练习系统。  
`V1` 的目标是完成“知识点录入 -> 训练出题 -> 回答评估 -> 掌握度更新 -> 历史查询”的最小可用闭环。

## 2. 当前技术栈

后端：

- Java 21
- Spring Boot 3.3.6
- Spring AI 1.0.0-M5
- Spring Web / Data JPA / Validation / Security / Redis
- PostgreSQL

前端：

- Vue 3
- TypeScript
- Vite
- Pinia
- Vue Router
- Tailwind CSS
- Axios
- ECharts

## 4. V1 核心业务规则

1. 知识点创建后默认 `mastery = 0`
2. 训练提交后，`overall` 按三维平均分计算
3. 掌握度公式：

```text
newMastery = oldMastery * 0.7 + overall * 0.3
```

4. 删除知识点时需要先删除关联训练记录
5. `TrainingRecord.suggestions` 在 API 返回中固定为 `string[]`

## 5. 当前公开接口

固定公开接口：

1. `GET /api/knowledge`
2. `GET /api/knowledge/{id}`
3. `POST /api/knowledge`
4. `PUT /api/knowledge/{id}`
5. `DELETE /api/knowledge/{id}`
6. `POST /api/training/start`
7. `POST /api/training/submit`
8. `GET /api/training/history/{knowledgeId}`
9. `GET /api/training/history`

默认 Base URL：

```text
http://localhost:8080/api
```

## 6. 当前配置结论

1. V1 当前推荐数据库方案：先使用 PostgreSQL 单库
2. MySQL 配置仍保留在配置体系中，但不是 V1 主落地路径
3. 敏感配置通过环境变量注入
4. Spring Boot 不自动读取 `.env`，需要通过脚本或 systemd 注入
5. 前端生产环境必须显式设置：

```env
VITE_API_BASE_URL=https://your-domain.com/api
```

## 7. 当前安全与可观测性结论

1. V1 不启用用户鉴权体系
2. 后端仅放行：
   - `/api/knowledge/**`
   - `/api/training/**`
3. 其他路径默认拒绝
4. 后端返回 `X-Request-Id`
5. 日志默认带 `requestId`
6. LLM 日志只记录长度、指纹、耗时，不记录原始 prompt/response

## 9. 已知风险与后续建议

仍需关注：

1. LLM 上游返回格式存在漂移风险
2. 当前未实现真正的用户鉴权与权限控制
3. 生产环境需要继续做好密钥管理与服务限流

后续自然方向：

1. 规划 V2 需求边界
2. 决定是否引入用户体系 / JWT
3. 决定是否引入知识库检索与多轮对话
4. 决定是否启用 MySQL + PostgreSQL 双库职责拆分
