# InterviewOS V3 归档

## 1. 归档说明

本文档用于冻结 InterviewOS `V3` 阶段的范围、交付物、验收结果和后续阅读入口。

归档原则：

- 将 `docs/plan` 下的 V3 文档迁移到 `docs/archive/v3/plan/`
- `docs/plan` 仅保留当前活跃版本 V4 文档
- 通过归档索引统一收敛 V3 的最终状态
- 后续如进入 V4 实施阶段，不再直接修改 V3 已冻结结论

归档时间：

- `2026-04-09`

V3 当前状态：

- 已完成
- 已完成文档收口与归档
- 可作为 V4 桌面化前端重构的稳定业务基线

## 2. V3 范围结论

V3 已完成能力：

1. 文件导入从单步落 Knowledge 升级为 document / chunk / embedding pipeline
2. 基于 PostgreSQL + pgvector 的最小可用检索能力
3. 训练题目、hint、feedback 接入 retrieval context
4. 训练引用快照与结果页引用展示
5. 自动知识点识别候选结构与最小前后端闭环
6. 导入进度、文档列表、检索验证页与引用展示的前端接入
7. 安全、幂等、可观测性的首轮补齐

V3 不包含：

1. 桌面化前端工作区
2. 全新的窗口系统与 Dock 导航模型
3. 独立的 `frontend-desktop` 应用
4. 移动端与桌面端双壳层策略
5. 全局命令面板、通知中心等桌面化交互

## 4. 代码与运行基线

运行基线：

1. 后端：Java 21 + Spring Boot 3.3.6
2. 前端：Vue 3 + TypeScript + Vite
3. 默认 API Base URL：`http://localhost:8080/api`
4. 当前主业务数据库：PostgreSQL + pgvector
5. V3 结束时主业务能力已完成文件级知识库、RAG 检索增强训练与候选知识点识别

## 5. 归档后的使用方式

如果要开启新的对话或新阶段工作，建议：

1. 先把 [V3 交接摘要](./V3交接摘要.md) 发给新的助手作为上下文
2. 如果涉及 V3 数据结构或接口兼容，再补充 [V3 数据库设计](./plan/V3数据库设计.md) 与 [V3 接口设计](./plan/V3接口设计.md)
3. 如果进入 V4，请优先阅读 `docs/plan/` 下的 V4 文档，而不是继续修改 V3 归档文件
