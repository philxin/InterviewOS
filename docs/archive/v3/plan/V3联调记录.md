## 4. 已验证链路

### 4.1 文件导入与文档索引

- 文件导入状态流：`PENDING -> PROCESSING -> CHUNKING -> EMBEDDING -> SUCCESS/PARTIAL/FAILED`
- 前端已展示 `totalChunks / embeddedChunks / failedChunks / PARTIAL`。
- 导入失败原因已做脱敏处理，避免直接回传内部堆栈。

### 4.2 文档与检索

- `/knowledge/documents` 列表与详情可用。
- `/rag/search` 可按当前用户隔离召回结果。
- 新增 `retrieval_trace` 落表，记录 `topK / hitCount / degraded / chunkIds / scoreDistribution`。

### 4.3 训练增强与引用快照

- 训练启动 / hint / feedback / session detail 均可返回 `retrievalMode + references`。
- 前端训练作答页与结果页已展示引用材料。
- 引用材料进入 prompt 前增加基础 `prompt injection` 过滤。

### 4.4 安全与回归

- 双用户检索不交叉（`RetrievalServiceTest`）。
- 非 owner 文档访问返回 `404`（`KnowledgeDocumentServiceTest`）。
- prompt injection 样例不会直接进入 LLM 引用上下文（`PromptInjectionGuardTest` + `TrainingSessionServiceTest`）。
- 导入失败原因不泄露堆栈（`KnowledgeFileImportProcessorTest`）。
- 已完成导入任务重复执行会短路跳过（`KnowledgeFileImportProcessorTest`）。
